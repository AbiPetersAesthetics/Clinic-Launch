I need to upgrade the Clinic Launch OS project plan and financial model so it tracks ACTUALS vs PLANNED spend.

Context:
The app currently shows a selected/planned project cost, for example £84,789. This is useful for forecasting, but I am now starting to spend real money on the project, so I need the model to track actual spend, committed spend, forecast final cost, and variance against plan.

The system should become a proper project controls dashboard, not just a planning estimate.

Please implement the following:

1. TASK-LEVEL ACTUALS

For every project task, add the following fields:

- Planned Cost
- Committed Cost
- Actual Cost
- Supplier
- Invoice Reference
- Invoice Date
- Paid Status
- Payment Date
- Notes

Definitions:

Planned Cost = the existing selected cost for the task.
Committed Cost = signed quote, invoice received, or cost committed but not yet paid.
Actual Cost = money actually paid.

Example:
Task: Architectural drawings / Parkers Design
Planned Cost: £2,903
Committed Cost: £2,903
Actual Cost: £2,903
Supplier: Parkers Design
Invoice Ref: 1593
Invoice Date: 04/06/2026
Paid Status: Paid

2. VARIANCE CALCULATIONS

For each task, calculate:

Variance £ = Actual Cost - Planned Cost

Variance % = ((Actual Cost - Planned Cost) / Planned Cost) x 100

If Actual Cost is £0 but Committed Cost exists, calculate committed variance as:

Committed Variance £ = Committed Cost - Planned Cost

Committed Variance % = ((Committed Cost - Planned Cost) / Planned Cost) x 100

Display both where useful.

Use traffic-light logic:

- Blue = under budget
- Green = on budget or within +5%
- Amber = +5% to +10% over planned
- Red = more than +10% over planned
- Grey = no actual or committed cost entered yet

4. FORECAST FINAL COST LOGIC

Forecast Final Cost should be calculated as:

Actual paid costs
+
Committed unpaid costs
+
Remaining planned costs for tasks not yet committed or paid

Do not simply add variance onto the whole budget.

For each task:
- If Actual Cost exists and Paid Status = Paid, use Actual Cost.
- Else if Committed Cost exists, use Committed Cost.
- Else use Planned Cost.

Then sum all tasks.
