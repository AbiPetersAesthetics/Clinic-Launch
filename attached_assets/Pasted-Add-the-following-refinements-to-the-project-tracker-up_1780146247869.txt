TASK 1 — ADD VAT HANDLING TO COST MODEL

Update the cost model so every task can clearly state whether costs are:

* ex VAT
* inc VAT
* VAT not applicable
* VAT unknown
* mixed / depends on supplier

If the app currently only stores one cost number, add one of the following:

Preferred option:

* costLowExVat
* costMidExVat
* costHighExVat
* costLowIncVat
* costMidIncVat
* costHighIncVat
* vatStatus

Alternative simpler option:

* costLow
* costMid
* costHigh
* costVatStatus
* costNotes

Use this logic:

If a quote is from a VAT-registered professional company, assume the quoted figure may be ex VAT unless clearly stated otherwise.

If a task cost is an estimate, mark VAT status as “VAT unknown / estimate”.

If costs are likely to include retail purchases, mark as “inc VAT estimate” unless trade pricing is expected.

If costs are professional fees, contractor works, design fees or commercial services, mark as “ex VAT unless confirmed”.

The dashboard should show both:

* Estimated cost excluding VAT
* Estimated cost including VAT

Where VAT status is unknown, include the cost in the “possible VAT exposure” calculation.

Add a dashboard warning:

“Some project costs may be ex VAT. Confirm VAT status before final budget lock. Use inc VAT figures for cashflow planning unless VAT recovery has been confirmed by the accountant.”
